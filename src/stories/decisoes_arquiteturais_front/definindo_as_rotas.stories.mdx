### Montando a estrutura de rotas:

<div>
  <span>
    O React Router é composto por uma <strong>série de componentes</strong>. Estes podemos
    usar da forma que nos for interessante no momento. Neste projeto escolhi fazer uso dos seguintes
    componentes:
  </span>
  <ul>      
    <li>
      <span><strong>Browser Router:</strong> Pra gerenciar a navegação entre as páginas num
        navegador web. Ele também permite acesso ao histórico de navegação e à manipulação da barra de endereços.</span>
    </li>
    <li>
      <span><strong>Route:</strong> Permite escolher qual componente renderizar de acordo com um
        caminho recebido.</span>
    </li>
    <li>
      <span><strong>Switch:</strong> É quem vai de rota em rota e verifica se o caminho dela
        corresponde ao que está na URL. Funciona como um Switch-case de rotas.</span>
    </li>
    <li>
      <span><strong>Link:</strong> Permite navegar para uma rota ao clicar nele.</span>
    </li>
  </ul>
</div>

### Definindo nomes:

<br />

<span>
  Definir um caminho estático nas rotas e links, faz com que sempre que precisarmos <strong>alterar</strong> este
  caminho, tenhamos que ir de <strong>arquivo em arquivo</strong> que redireciona pra essa rota, e lembrar de alterar o
  caminho da rota.
</span>

<br />
<br />

<span>
  Uma forma simples de resolver isso é definir um <strong>nome</strong> para aquela rota, e apontar pro nome, visto que
  ele <i>nunca</i> muda.
</span>

<br />

### Rotas diferentes no front-end e no back-end:

<span>
  Uma vez que o Front-End é exposto para o usuário final, decidimos ter sempre rotas na linguagem do usuário e
  entendíveis.
</span>
<span>
  Porém no Back-End mantivemos a estrutura solicitada.
</span>
