### Considerações sobre o uso do Create React App:

<div>

  <ul>
    <li>
      <span>Quando escolhemos usar o Create React App, estamos escolhendo a decisão arquitetural de utilizar as
        <strong><i>configurações</i></strong> que já vêm junto com ele por causa do <strong>React
          Scripts</strong>.</span>
    </li>
    <li>
      <span>É considerada uma decisão arquitetural, pois uma vez que usamos o React Scripts, não precisamos configurar o
        Webpack, nem o Babel. <strong>Porém isso também dificulta que façamos configurações customizadas no Webpack ou
          no Babel.</strong></span>
    </li>
  </ul>
</div>

<br />

### Gerando a versão de produção do projeto:

<br />

<span>
  Quando decidimos rodar um projeto React em ambiente de produção, precisamos executar uma série de tarefas para
  otimizar o projeto para rodar bem nos navegadores dos usuários consumindo o mínimo possível da sua internet e do nosso
  servidor.
</span>

<br />
<br />

### Algumas dessas tarefas incluem:

<div>
  <ul>
    <li>
      <span>Minimizar os arquivos javascript e css.</span>
    </li>
    <li>
      <span>Transpila seu código ES6 ou posterior para ES2015.</span>
    </li>
  </ul>
</div>

<div>
  <span>
    Por ter escolhido usar o Create React App, já temos tudo isso configurado no comando <strong>build</strong> que
    o React Scripts nos disponibiliza.
  </span>

  <br />
  <br />

  <div className="flex">
    <code>npm run build</code>
    <code>yarn build</code>
